RFC-014 · IN REVIEW IN REVIEW
Web wallet architecture
IV Ivan Kalyaev · edited 2026-06-29
Summary

SpaceNotes should ship a first-party web wallet so a space can hold and sign for shared assets without a browser extension. We scope v1 to EIP-1193 providers1 and defer hardware signing.

Motivation

Today every space that touches on-chain data leans on MetaMask. That couples our UX to a third party and blocks mobile entirely.

  • —No extension required, including on mobile.
  • —One signing flow shared across every space.
  • —Auditable, per-space permissions.
i This RFC supersedes RFC-009 · Simplified accounts.
Proposed design

A thin provider shim implements request() and routes signMessage2 through the SpaceNotes signer service.

provider.ts 
provider.request({
  method: "personal_sign",
  params: [msg, account],
})
